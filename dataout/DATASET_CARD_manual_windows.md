# Dataset Card: Manual Windows Dataset (02.1)

## 1. Summary

- **Name:** Oil Spill Detection Dataset - Manual Windows build
- **Version:** v0.1.0 (working)
- **Primary task:** oil spill segmentation / detection using manually adjusted windows

- **Generation references:**
	- `02.1.1-Estimate_windows_over_geoms_by_target_size.py`: Do a first shot in windows positioning over the geometries of oil spills/seapage.
	- `02.1.2-Adjust_estimate_windows.py`: Tools to adjust the position of the windows generated by 02.1.1.
	- `02.1.3-Build_DS_from_manual_windows.py`: Build the image dataset from the adjusted windows positioning from 2.1.2.
- **Dataset root:** `02.1-DS_by_manual_windows/800x600_windows/`

## 2. Data Composition

- **Pipeline stages:**
	- `1/` estimated windows and visualization artifacts (`Created_windows.json` + per-image PNG overlays)
	- `2/` adjusted windows and review artifacts (`Adjusted_windows.json` + global/current focus plots)
	- `3/` final dataset payload (`CSV/`, `IMAGES/`, `RASTER/`)
- **Primary feature table:** `3/CSV/Oil_Stats_manual_windows_16bits_800x600.csv`
- **Rows in manual-windows CSV:** 44
- **Class distribution (CLASSE):**
	- `OIL SPILL`: 36
	- `SEEPAGE SLICK`: 8
- **Image/mask bundles in final stage:**
	- `3/IMAGES/IMG-RGB/`
	- `3/IMAGES/LABELS-1D/`
	- `3/IMAGES/LABELS-RGB/`

## 3. Build and Validation Workflow


- **Window estimation stage:** candidate windows are created from source geometries and target window size settings (`Created_windows.json`).
- **Manual adjustment stage:** window placement is reviewed and refined into a canonical adjusted set (`Adjusted_windows.json`).
- **Final materialization stage:** the curated windows are converted into CSV, image-mask triplets, and raster outputs.
- **Triplet validation reference:** `03-Validate_image_triplets.py`
	- Recommended root for this dataset: `02.1-DS_by_manual_windows/800x600_windows/3/IMAGES`

## 4. Feature Scope

The final CSV follows the same broad descriptor families as the geom-bbox dataset, including:

- geometry/shape descriptors
- Hu moments
- foreground/background radiometric descriptors
- class labels (`CLASSE`)

## 5. Relationship to 02.0 Dataset

- The 02.0 dataset is bbox-driven and larger (385 rows in the 16-bit CSV).
- The 02.1 dataset is manually curated for window placement and smaller (44 rows in the 16-bit CSV).
- Both are useful: 02.0 for scale and 02.1 for more controlled windowing.

## 6. Known Limitations

- Manual-window process is more labor-intensive and currently tied to the configured target size branch (`800x600_windows`).
- Class imbalance remains present in the current generated CSV.
- License/citation metadata still require final publication values.
