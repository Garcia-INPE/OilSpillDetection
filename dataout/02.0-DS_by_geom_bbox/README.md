# Oil Spill Dataset (DS)

This folder contains a publish-ready package structure for oil spill detection research.

## Structure

- `CSV/` tabular features and statistics.
- `IMAGES/` original image and mask assets.
- `splits/` train/validation/test manifests.
- `DATASET_CARD.md` dataset documentation for publication.
- `checksums.sha256` file integrity registry.

## Image bundles

- `IMAGES/IMG-RGB/` original RGB image crops.
- `IMAGES/LABELS-1D/` original single-channel label masks.
- `IMAGES/LABELS-RGB/` original RGB label masks.
- `IMAGES/validation_report.csv` validation report generated by [03-Validate_image_triplets.py](../../../03-Validate_image_triplets.py).

## Validation

- Triplet validation checks pairing across `IMG-RGB`, `LABELS-1D`, and `LABELS-RGB`.
- By default, validation checks consistency of dimensions per sample.
- Optional fixed-size enforcement can be enabled with `--width` and `--height`.

## Required conventions

- Every sample must have a unique `sample_id`.
- Naming should use one convention only (recommended: lowercase + underscore).
- Relative paths in split files must be rooted at this `DS/` directory.

## Minimal publication checklist

1. Fill `DATASET_CARD.md` with final metadata and citation.
2. Populate split manifests in `splits/`.
3. Generate checksums into `checksums.sha256`.
4. Ensure license and usage rights are clearly stated in the dataset card.
5. Run [03-Validate_image_triplets.py](../../../03-Validate_image_triplets.py) to validate image/label triplets in `IMAGES/`.

## Notes on empty directories

Empty folders are acceptable placeholders if described in this README and the dataset card.
If a modality is not available, keep the folder and mention "currently empty" in `DATASET_CARD.md`.
